# 第3章 系统与方法设计

## 3.1 问题定义

面向多域光网络自治运维场景，本文聚焦于两类典型风险：链路拥塞（High Congestion）与光信号质量劣化（Low QoT）。给定网络拓扑、链路属性与业务请求集合，系统目标是在保证业务连续性的前提下，自动选择控制动作以实现风险缓解与性能优化。本文将控制目标形式化为多目标优化问题，核心关注以下指标：  
1) 风险修复能力（risk repair rate）；  
2) 业务 QoT 指标（平均 OSNR、低 QoT 风险残留）；  
3) 控制开销（重路由次数、控制消息数量、路径代价）。

与传统单控制器方案相比，光网络风险处置往往具有跨层信息异构、决策时效要求高、策略稳定性与可解释性并重等特点。为此，本文采用“多智能体协同控制 + 轻量强化学习决策”的技术路线：系统层面使用多智能体闭环结构组织信息流与控制流，算法层面在决策智能体引入轻量 DQN，并与规则法、Q-Learning 基线对比。

## 3.2 多智能体协同架构

本文系统采用基于 OODA 思想的多智能体协同闭环，包括感知（Perception）、意图解析（Intent）、证据检索（Evidence）、决策（Decision）、执行（Action）与评估（Evaluation）六个功能体。结合工程可落地性与论文叙事收敛需求，本文将其划分为“核心三智能体 + 辅助模块”的组织方式：

- 核心智能体  
  1) 感知智能体：识别拥塞与低 QoT 风险，维护关键状态量；  
  2) 决策智能体：依据状态选择控制动作；  
  3) 执行智能体：执行路径切换或功率调节并更新网络状态。

- 辅助模块  
  1) 意图解析模块：将风险报告转为结构化意图；  
  2) 证据检索模块：从本地知识库检索规则证据；  
  3) 评估反馈模块：根据执行前后状态变化计算奖励并更新策略。

该结构兼顾了系统可解释性与策略可优化性：感知与执行维持光网络物理语义约束，决策层负责策略学习，评估层提供闭环反馈，使系统在不依赖重型训练框架的情况下具备在线策略改进能力。

## 3.3 决策方法设计

本文在同一控制框架下实现三类决策策略：

1. 规则法（Rule）  
基于风险类型的启发式策略：拥塞优先重路由，低 QoT 优先功率提升。该方法计算开销低、可解释性强，作为工程基线。

2. 表格型 Q-Learning  
将状态映射为离散标签，使用 Q 表执行动作选择与值更新。该方法在中高负载下体现了较好的稳定性，作为强化学习传统基线。

3. 轻量 DQN（主方法）  
采用两层 MLP 逼近 Q 函数，使用 Experience Replay 与 Target Network 提升训练稳定性。该方法在低/中负载及 QoT 指标上表现出更强优化能力。

三类方法共享统一控制语义，本文在最终配置中采用参数化动作空间：  
- `Action_Maintain`：保持现状；  
- `Action_Power_Boost_1p0dB / 2p0dB / 3p0dB`：不同档位功率提升；  
- `Action_Reroute_K1 / K3 / K5`：不同候选范围的重路由。  

该设计使决策层不仅能选择“做什么动作”，还能选择“动作强度/范围”，在风险修复能力与控制开销之间实现更细粒度折中。

## 3.4 状态空间与奖励设计

### 3.4.1 状态空间

为避免状态维度过高导致训练不稳定，本文构建低维连续状态向量，主要包含：业务等级、风险类型、带宽归一化、路径长度、跳数、OSNR 裕量、路径最大链路利用率、候选路径数量。该设计遵循“物理可解释 + 决策相关”的原则，确保特征既反映网络运行状态，又与动作效果直接关联。

### 3.4.2 连续型 KPI 奖励函数

相比固定离散奖励，本文采用“执行前后差值驱动”的连续奖励机制。对低 QoT 业务，奖励由 OSNR 改善幅度、是否达标、动作代价构成；对拥塞业务，奖励由链路利用率下降幅度、是否回落阈值、重路由失败惩罚构成。对核心业务重路由额外施加保护惩罚，以体现工程风险控制约束。

该奖励机制能更真实地反映网络优化目标，避免仅依据动作类别进行粗粒度打分，从而提高策略学习的有效性与可解释性。

# 第4章 实验设计与结果分析

## 4.1 实验设置

### 4.1.1 数据与环境

实验基于 TEFNET 拓扑与流量数据，在 Python 环境下完成仿真。系统按轮次从业务全集中抽取活跃业务，执行完整闭环。决策层分别采用 Rule、Q-Learning、DQN，并在统一流程下比较性能。

### 4.1.2 训练与评估配置

主实验采用“训练-评估分离”策略：训练阶段进行策略更新，评估阶段固定策略输出，以降低在线学习噪声。典型配置如下：训练轮数 120，评估轮数 30，采用多随机种子（42/52/62）进行复验。

### 4.1.3 指标定义

本文使用以下指标评估方法性能：

- `risk_repair_rate`：风险修复率，衡量整体风险处置能力；
- `risk_after`：执行后剩余风险数，越低越好；
- `low_qot_after`：剩余低 QoT 风险数，衡量信号质量恢复效果；
- `avg_osnr_after`：执行后平均 OSNR，越高越好；
- `congestion_after`：执行后拥塞风险残留；
- `avg_reroute_actions`、`control_messages`：控制开销指标。

## 4.2 主对照实验结果

在中负载场景的多随机种子复验中，DQN 与 Q-Learning 结果接近，且在不同版本配置下表现存在波动。优化后 DQN 在综合修复率上可实现对 Q-Learning 的小幅领先，同时在低 QoT 风险消减和平均 OSNR 上保持优势。

从结果可见，DQN 的主要收益来源于 QoT 维度优化，其策略更倾向于主动动作选择；相应地，控制开销可能上升，体现出“性能提升与开销成本”的工程权衡关系。

## 4.3 负载分层实验结果

为评估方法在不同业务压力下的鲁棒性，本文设置低/中/高三档负载（每轮活跃业务数分别为 10、15、25）并进行多随机种子复验。结果显示：

1. 低负载场景：DQN 在风险修复率、低 QoT 风险残留和平均 OSNR 上均优于 Q-Learning；  
2. 中负载场景：DQN 与 Q-Learning 接近，DQN 在 QoT 指标上保持小幅优势；  
3. 高负载场景：Q-Learning 在总体修复率上更稳健，DQN 优势减弱。

该结果表明，当前 DQN 策略更适用于中低负载下的质量优化任务，而在高负载拥塞场景中仍需进一步增强拥塞导向能力。

## 4.4 DQN 消融实验

为验证 DQN 关键机制的贡献，本文设置以下消融项：  
1) 去除 Replay/Target 机制；  
2) 去除引导式探索策略。  
在相同随机种子和中负载条件下，消融结果显示：

- DQN 配置对训练稳定性较敏感；  
- 去除某些机制后局部指标可能短期提升，但波动增大；  
- 全量机制配置（DQN_Full）在整体指标与稳定性上更均衡，适合作为最终方案。

该实验说明了“仅看单一指标最优”并不足以反映工程可用性，稳定性与一致性同样是关键考量。

## 4.5 扩展探索：VDN-lite

本文进一步尝试了价值分解式扩展（VDN-lite），将决策拆分为路由子智能体与功率子智能体并进行联合价值估计。实验显示该扩展在部分场景具备改进潜力，但在当前算力与样本规模约束下，尚未稳定超过主基线（Q-Learning 与优化后 DQN）。因此，VDN-lite 在本文中定位为扩展探索，而非主方法。

## 4.6 本章小结

本章实验结果说明：在统一多智能体协同框架下，轻量 DQN 方案能够在中低负载场景下提升 QoT 相关指标，并在综合修复能力上达到与传统方法可竞争甚至小幅领先的水平；同时，该方法在高负载拥塞场景下仍存在稳定性与拥塞导向能力不足的问题。后续工作应重点面向高负载鲁棒性优化与更稳定的多智能体价值分解训练机制。

# 第5章 结论与展望（草稿）

## 5.1 结论

本文围绕光网络自治运维问题，设计并实现了多智能体协同控制系统，在统一框架下对 Rule、Q-Learning 与 DQN 方法进行了系统比较，并通过多随机种子复验、负载分层实验和消融实验验证了方法特性。研究表明：  
1) DQN 在中低负载和 QoT 指标上具有优势；  
2) Q-Learning 在高负载场景下表现出更好的稳定性；  
3) 系统性能提升伴随控制开销增加，需在收益与代价间做工程折中。

## 5.2 展望

后续可从以下方向进一步改进：  
1) 强化拥塞导向奖励与约束机制，提升高负载场景性能；  
2) 引入更稳定的多智能体价值分解训练框架（如改进版 VDN/QMIX）；  
3) 完善在线部署约束与控制消息优化，提升工程可落地性。
