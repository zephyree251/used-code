# 中期反馈整改说明（聚焦 multi-agent + 光网络）

## 一、问题复盘

中期反馈指出：当前工作存在进展滞后、研究点分散、缺少对具体网络算法聚焦的问题。

## 二、整改目标

在不偏离题目“基于 multi-agent 的光网络智能控制研究与设计”的前提下，将后续工作收敛为：

- 研究对象：光网络中的拥塞与低 QoT 风险协同处置
- 研究方法：多智能体闭环（Perception/Intent/Evidence/Decision/Action/Evaluation）
- 研究重点：决策算法（Rule vs Q-Learning vs 轻量 DQN）在重路由与功率调节上的性能差异

## 三、技术聚焦点

1. **网络算法聚焦**  
   以 `K-Shortest Paths (KSP)` 候选路径搜索 + 动作选择（重路由/调功率/保持）为核心。

2. **多智能体定位清晰**  
   - 感知智能体：识别拥塞与低 QoT 风险  
   - 决策智能体：输出动作策略（规则或 RL）  
   - 执行与评估智能体：实施动作并回写奖励，形成闭环学习

3. **实验可复现**  
   新增 `benchmark_focus.py`，在相同业务采样下完成 Rule、Q-Learning 与轻量 DQN 的对照测试。

## 四、量化评估指标

- 风险修复率（执行前后风险数量变化）
- 拥塞风险减少量
- 低 QoT 风险减少量
- 平均 OSNR 提升
- 重路由次数与功率调节次数（动作开销）

## 五、后续四周计划

- 第 1 周：完成实验参数固化（样本规模、轮数、随机种子）与 baseline 校验
- 第 2 周：完成 Rule vs RL 多组负载实验并导出结果表
- 第 3 周：完成结果分析与消融讨论（拥塞场景/低 QoT 场景）
- 第 4 周：完成论文实验章节与结论章节撰写

## 六、预期成果

- 一套可复现实验脚本与指标结果
- 一组支撑“multi-agent + 光网络控制有效性”的对照证据
- 一条与题目高度一致、可答辩的研究主线
